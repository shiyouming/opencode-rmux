# opencode-rmux 入门指南

> 你不用学任何命令。装上插件后，直接跟 AI 聊天，AI 帮你操作终端。

---

## 准备工作

这个插件让你在 AI 对话框里直接指挥终端窗口。但它依赖一个叫 **RMUX** 的终端管理工具（类似 tmux）。

**没装 RMUX？** 让 AI 帮你装：

```
帮我安装 RMUX
```

AI 会根据你的系统（Windows / macOS / Linux）给你安装命令。

**装了但不认识它？** 没关系，你不需要直接使用 RMUX，AI 会帮你操作它。

---

## 快速体验（2 分钟）

确保 RMUX 已安装、opencode 已重启后，在对话框里对 AI 说：

```
帮我看看现在终端里有哪些窗口在跑
```

如果 AI 回答了你（比如"有一个 opencode 会话"），说明插件已生效。就这么简单。

如果 AI 说不知道或报错，换句话试试：

| 你想说 | 可以这么说 |
|--------|----------|
| 看有什么窗口 | "看看现在有哪些终端" / "我的终端里有什么" |
| 看某个面板 | "看看右边那个面板在跑什么" |
| 操作面板 | "帮我查查 3000 端口" / "帮我跑个命令" |
| 等结果 | "等它跑完告诉我" |

## 你能让 AI 帮你做什么

### 1. 看看终端里有什么

```
看看现在有哪些终端会话
右边那个面板在跑什么程序
帮我找找哪个面板在跑 npm
```

AI 会帮你扫描所有窗口和面板，告诉你哪个在跑什么。

你看到输出里类似 `opencode:0.1` 的东西，那就是面板的地址——你不需要记，AI 自己会用它。

### 2. 让 AI 帮你读屏幕

```
看看那个面板现在显示什么
我那个构建跑完了吗
帮我看一下报错信息
```

AI 会读取面板上的文字内容，然后告诉你结果。你不用自己切过去看。

### 3. 让 AI 帮你操作其他面板

```
在右边那个面板里帮我跑一下 npm install
帮我查查 3000 端口被谁占了
帮我把最新代码部署上去
```

AI 会在目标面板里输入命令并执行。命令跑完后，你的键盘焦点自动回到主界面，你接着聊天就行。

### 4. 让 AI 帮你盯着，完事了告诉你

```
帮我启动前端 dev server，启动好了告诉我
等构建完成通知我
帮我盯着编译，有报错马上说
```

AI 会发命令启动，然后持续观察输出，直到看到你关心的内容（比如"编译成功"）才通知你。

### 5. 同时看多个面板的输出

```
帮我同时看看前端和后端的日志
为什么构建这么慢，把每步的输出都给我看看
```

AI 会同时监听多个面板的输出，汇总给你看。

### 6. 创建新会话

```
帮我开一个独立的窗口跑 htop 监控系统
帮我建一个新的会话用来调试
```

AI 会创建一个新的独立会话，在里面运行你指定的程序。

### 7. 子代理工作区 — 全自动，不需要你说话

当 AI 派出子代理去并行查资料时，你会在右侧看到新面板自动出现：

```
+----------+------+
|          |子代理|
|  主对话  |正在工|
|   面板   |作   |
+----------+------+
```

多个子代理同时干活时，右侧会堆叠多个面板：

```
+----------+------+
|          | 子1  |
|  主面板  |------|
|          | 子2  |
|          |------|
|          | 子3  |
+----------+------+
```

子代理做完一个，面板就自动关一个，你只管在主面板等结果。

## 完整的真实场景

### 场景：排查问题

```
你：我的 Web 服务报端口被占了，帮我查查怎么回事
```

AI 会自动：

1. 找哪个面板在跑你的服务
2. 看面板上有什么报错信息
3. 查端口被谁占着
4. 告诉你结果和处理建议

你只说了第一句，后面全是 AI 自己做的。

### 场景：自动化部署

```
你：帮我把最新代码部署上去
```

AI 会一步步来：拉代码 → 构建 → 启动 → 通知你完成。整个过程你只用说一句，AI 帮你盯着每一步的结果。

## 如果 AI 没理解你的意思

换个说法试试：

| 你想做的事 | 可以这么说 |
|-----------|----------|
| 看有什么窗口 | "看看现在有哪些终端" |
| 看某个面板 | "看看右边那个面板" |
| 发命令 | "在那个面板里跑一下 X" |
| 等结果 | "等它跑完告诉我" |
| 看日志 | "帮我持续看看它的输出" |

万一还是不行，直接说你想看的现象：
```
帮我看看右边那个面板屏幕上有什么
```
AI 就知道该调哪个功能了。

## 要不要改设置？

默认情况下插件直接可用。如果你想调整行为，在电脑上建个文件：

**Windows：**
```
%USERPROFILE%\.config\opencode\opencode-rmux.json
```
（相当于 `C:\Users\你的用户名\.config\opencode\opencode-rmux.json`）

**macOS / Linux：**
```
~/.config/opencode/opencode-rmux.json
```

```json
{
  "splits": true,
  "splitSize": "30%",
  "keepPaneOnIdle": false,
  "maxPanes": 4,
  "notifications": {
    "done": true,
    "error": true
  }
}
```

| 选项 | 默认 | 说明 |
|------|------|------|
| `splits` | `true` | 子代理是否自动分屏 |
| `splitSize` | `"30%"` | 分屏宽度 |
| `keepPaneOnIdle` | `false` | 完事后是否保留面板 |
| `maxPanes` | `4` | 最多几个子代理同时显示 |
| `notifications.done` | `true` | 完成时是否弹通知 |
| `notifications.error` | `true` | 出错时是否弹通知 |

改完后重启 opencode 生效。
